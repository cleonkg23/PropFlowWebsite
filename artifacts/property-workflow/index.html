The work is not hard to understand. It is just too easy to lose.

  2. ii.A viewing happens, but nobody follows up.
  3. iii.A landlord asks for an update and the answer is buried in someone’s inbox.

§ 02 — What PropertyFlow helps with

What PropertyFlow helps with

Tenant and landlord messages

Inbox email → named task with owner and draft reply. No thread digging.

Maintenance jobs

Repair request → task assigned to team member or contractor, tracked to completion.

Viewing follow-ups

Each viewing gets a follow-up task. The board flags it before the applicant has to chase.

Document chasing

References, ID, deposits, signed agreements — all tracked in one place until everything is in.

Contractor updates

Jobs waiting on a contractor are visible at a glance. No ringing around to check.

Manager visibility

Open, overdue and completed — one view for the whole team. No status meetings needed.

Your tags, categories and statuses — you define them. PropertyFlow is set up around your actual workflow, not a generic template.

§ 02b — Where this fits

This does not replace your existing property software.

Keep Arthur, Re-Leased, Fixflo, Alto or whatever you use. PropertyFlow handles the work that falls between them: emails, follow-ups, contractor updates and internal handovers.

  • −
    Arthur / Re-Leased / AltoProperty records, tenancies and management — keep using them.
  • −
    FixfloMaintenance reporting — keep using it. PropertyFlow handles what happens after the report.
  • −
    Front / Help ScoutShared inbox — not built around properties, contractors or repair timelines.
  • +
    PropertyFlowTurns property work into assigned tasks with draft replies, owners and status tracking. Shaped around the work small property teams already do every day.

§ 04 — How it works

How it works

  1. 01
    Pick one messy workflow

    Choose the process causing the most admin: repairs, viewings, landlord updates or document chasing.

  2. 02
    Map how it works today

    We look at where the work currently lives — inboxes, spreadsheets, WhatsApp, CRMs and contractors.

  3. 03
    Configure the board

    Set up your own categories, owners, statuses, reminders and draft reply templates. Everything is yours to define.

  4. 04
    Run it on real work

    Use it alongside your current tools. No big switchover. No disruption to the team.

  5. 05
    Add more workflows when it is working

    Start with one. When the first workflow runs cleanly, add the next one.
