Build me a lightweight, static, single-page marketing website for my workflow automation service for small property businesses in the UK.

This site is intended to be hosted on GitHub Pages, so you must respect GitHub Pages constraints from the start.

NON-NEGOTIABLE TECH CONSTRAINTS
- Use only plain HTML, CSS, and vanilla JavaScript.
- Do not use React, Next.js, Vue, Svelte, Tailwind, Bootstrap, jQuery, or any heavy framework.
- Do not use a backend.
- Do not use server-side rendering.
- Do not use authentication, databases, APIs, forms that require backend handling, or any build step that is necessary to run the site.
- The final site must be deployable by uploading static files directly to a GitHub repository.
- Include a .nojekyll file in the root so GitHub Pages serves the site cleanly as a static site.
- Use relative paths for all assets, CSS, JS, images, and links so the site works both on:
  1. a root GitHub Pages domain like username.github.io
  2. a project site like username.github.io/repository-name/
- Keep it fast and lightweight:
  - minimal JavaScript
  - no unnecessary animations
  - no large image libraries
  - no video backgrounds
  - no heavy font dependencies
  - prefer system fonts
  - use SVG where sensible
- Make the code clean, modular, readable, and easy for me to edit later.
- Output only static front-end files.

This is not a SaaS app. It is a lean client acquisition website for my service business.

The purpose is to help small estate agents and property managers quickly understand:

3. what workflows I can automate
4. what the demo shows
5. why they should contact me
